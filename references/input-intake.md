# Input Intake

Classify the source before generating slides.

## Input types

- `note`: rough notes, study notes, outlines. First rebuild a learning spine.
- `article`: prose article or explanation. Extract claims, mechanisms, examples, and sequence.
- `report`: business, research, or market report. Extract thesis, evidence, variables, risks, and implications.
- `technical-doc`: architecture, API, framework, code, or workflow document. Extract modules, state, interfaces, flow, failure modes.
- `existing-html`: existing slide deck. Preserve sequence, extract scene content, then restyle and deepen.
- `outline`: high-level headings. Expand into mechanism-level teaching content.
- `raw-ideas`: unordered fragments. Organize into premise, primitives, mechanisms, practice, and summary.
- `style-reference`: user-provided visual reference. Treat it as vocabulary, not a template to copy.

## Intake output

Record in `first-principles.md`:

